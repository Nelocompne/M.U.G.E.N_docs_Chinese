介绍

AIR是一种文本文件格式，用于描述一组动画。AIR格式在整个M.U.G.E.N中被广泛使用，用于对角色，背景，生命条等进行动画处理。

动画（在M.U.G.E.N中特别称为动画动作）描述了要显示的单个精灵（sprites）序列。角色的动画文件（.air）可以包含尽可能多的动画动作。

格式

这是.air文件中的典型条目。 接下来，我们将说明格式。

; Standing Animation
[Begin Action 000]
Clsn2Default: 2
 Clsn2[0] = -10,  0, 10,-79
 Clsn2[1] =  -4,-92,  6,-79
0,1, 0,0, 7
0,2, 0,0, 7
0,3, 0,0, 7
0,4, 0,0, 50
0,5, 0,0, 7
0,6, 0,0, 7
0,7, 0,0, 7
0,8, 0,0, 60

使用分号（;）进行注释。它们可以在任何地方使用，并且在同一行中的分号之后的所有内容都将被忽略。

[Begin Action n]
这定义了一个动作。 数字n称为动作编号。 每个动作必须具有唯一的动作编号，以便与其他动作区分开。 您可以有许多操作，但是请记住，有保留的操作编号。 这些数字及其相关的操作在下面列出。
Clsn2Default: 2
这表示正在定义两个碰撞箱，它们将为Clsn定义下面的所有条目定义。在这种情况下，对于这个动作中的每个精灵。如果你不想有一个默认的碰撞箱，那么将Clsn2Default改为Clsn2。你必须把Clsn定义放在精灵条目之前。每个精灵条目都可以有自己的碰撞箱。Clsn2指的是普通的碰撞框，Clsn1指的是 "attacking"框。你在进行攻击动作（如拳打脚踢或特殊动作）时使用攻击框。要定义碰撞箱，使用AirEdit，一个包含在M.U.G.E.N开发工具集的程序。
0,3, 0,0, 7

看起来像这样的条目称为"动画元素（animation element）"，或简称为"元素（element）"。 第一个数字是精灵组编号。 第二个数字是精灵图像编号。 制作 .sff 文件时，可以将这些数字分配给精灵。

因此，组＃0和图像＃3是使用sprmaker定义的，其条目为0,3, mypic.png, 40, 40.

元素的第3和第4个数字表示相对于精灵位置的X和Y偏移。 例如，对15,-10将使子画面的位置向前移动15个像素，并向上移动10个像素。 此偏移受缩放操作（scaling operations）影响，但不受旋转（rotation）影响。

第5个数字是在移动到下一个元素之前显示元素的时间长度，以游戏标记为单位。 以正常游戏速度在一秒钟内有60个游戏滴答声（game-ticks）。 对于第5个数字，如果希望该元素无限期显示（或直到切换到其他操作），则可以指定"-1"。 如果选择执行此操作，则仅在操作的最后一个元素上执行此操作。

动作的"循环时间（looptime）"是每个元素的所有时间的总和。 基本上，这是游戏提示中动作的总长度。 如果最后一个元素的时间为"-1"，则循环时间将视为无限。 我们称其为"no loop"。 具有有限循环时间的动作将在显示完最后一个元素后返回到第一个元素，并保持循环播放，直到您切换到另一个动作为止。

在上面的示例操作中，循环时间为： 7 + 7 + 7 + 50 + 7 + 7 + 7 + 60 = 152 ticks

; Standing Animation
[Begin Action 000]
Clsn2Default: 2
 Clsn2[0] = -10,  0, 10,-79
 Clsn2[1] =  -4,-92,  6,-79
0,1, 0,0, 7
0,2, 0,0, 7
Loopstart
0,3, 0,0, 7
0,4, 0,0, 50
0,5, 0,0, 7
0,6, 0,0, 7
0,7, 0,0, 7
0,8, 0,0, 60

用单词Loopstart添加一行将使动画从下一行的元素开始循环播放。 在上面的示例中，当显示Sprite 0,8的元素时，下一个要显示的元素将是Sprite 0,3的元素。

在测试动画时，您可能会发现暂停和帧前进功能很有用。 要暂停，请点击"Pause"按钮。 要按一个游戏滴答（game-tick）进行游戏，请按Scroll Lock。 进帧（Frame-advance）仅在游戏暂停时有效。

可选参数

对于这个条目：

15,1, 0,0, 5

如果要 水平和/或垂直 翻转精灵，则需要使用"翻转（flip）"参数：V代表垂直翻转，H代表水平翻转。 这些参数将排在第6位。 例如：

15,1, 0,0, 5, H   ;<-- 水平翻转
15,2, 0,0, 5, V   ;<-- 垂直翻转
15,3, 0,0, 5, VH  ;<-- 这将双向翻转，即。旋转180度

对于某些击打火花，您可能希望使用颜色附加来绘制精灵，使其具有透明效果。 这称为透明度混合。 颜色加法和减法的参数分别是A 和 S，并且应作为第7行。 例如：

15,4, 0,0, 5, ,A   ;<-- 颜色添加（省略翻转参数）
15,4, 0,0, 5, H, S ;<-- 水平翻转并进行颜色减法

If you wish to specify alpha values for color addition, use the parameter format AS???D???, where ??? are replaced with values of the source and destination alpha respectively. Values range from 0 (low) to 256 (high). For example, AS64D192 stands for "Add Source_64 to Dest_192". Also, AS256D256 is equivalent to just A. A shorthand for AS256D128 is A1.

15,4, 0,0, 5, ,A1  ;<-- 为目标颜色添加到50%深色
15,4, 0,0, 5, ,AS128D128 ;<-- Mix 50% source with 50% dest

要缩放精灵，请分别使用x比例和y比例的第8和第9个参数，例如，这将分别将精灵缩放1.5和2：

15,4, 0,0, 5, ,A, 1.5,2

要绕其轴旋转精灵，请在第10个参数中以度为单位指定角度：

15,4, 0,0, 5, ,A, 1,1, 45

注：1.1之前的M.U.G.E.N版本中不使用比例和角度参数。

插值

某些参数可以从一个元素插值到下一个元素。插值支持的参数有偏移、混合、比例和角度。

M.U.G.E.N 1.1及更高版本支持插值。

目前只支持线性插值。

语法：

Interpolate <type>

where <type> is one of Offset, Blend, Scale or Angle.

Offset
插值偏移。
Blend
插值透明度混合参数。两个元素必须具有相同的透明度函数，即 A. 要对完全不透明的元素进行插值，请使用 AS256D0.
Scale
插值比例。
Angle
插值角度。

可以在两个元素之间插入一条或多条插值（Interpolate）线，以便在它们之间进行插值。在循环动画中，第一个元素之前的插值（Interpolate）线将用于从最后一个元素插值到第一个元素。

以下示例将插值0,0和100,0之间的偏移，以及1,1和1.5,1.5之间的比例：

Interpolate Offset
Interpolate Scale
20,0, 0,0, 60, , , 1,1
Interpolate Offset
Interpolate Scale
20,0, 100,0, 60, , , 1.5,1.5

以下示例为旋转对象设置旋转周期为60个刻度的动画：

20,0, 0,0, 59, , , 1,1, 0
Interpolate Angle
20,0, 0,0, 1, , , 1,1, 354

建议行动编号

你不必完全遵循这一点，但它应该作为指导方针。一般来说，CNS中的状态应该与它们使用的动画具有相同的数字，以减少混淆。

您可能需要留出动画和状态编号的空间。这为您提供了根据需要添加更多操作的空间（某些攻击可以使用多个状态和动画）。例如，站立轻踢（Standing Light Kick）和站立强踢（Standing Strong Kick）的动作编号分别为200和210。

Number Description
0-199 reserved
200-299 Standing attacks
300-399 More standing attacks, running attacks
400-499 Crouch attacks
500-599 More crouch attacks
600-699 Air attacks
700-799 More air attacks
800-999 Unused - use if you need more states
1000-2999 All special attacks
3000-4999 All hyper attacks
5000-5999 reserved
