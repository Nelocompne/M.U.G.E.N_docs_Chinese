概述

M.U.G.E.N默认为英语程序，但支持切换游戏中显示的语言的功能。为了使主题和字符支持针对特定语言正确显示，必须为这些主题和字符中的每个添加和配置适当的语言字符串。配置新语言的过程称为本地化。

可以本地化的资产示例包括字符的胜利引号或标题屏幕中显示的标题徽标。

在M.U.G.E.N中，所需的语言是在mugen.cfg中配置的。 "语言"设置位于[Config]组中。当启动M.U.G.E.N运行时时，语言设置将被加载并在程序运行期间使用。

[Config]
Language = "en"

语言参数是ISO 639-1代码，它是与特定语言相对应的两个字母的代码。例如，要将首选语言设置为日语：

[Config]
Language = "ja"

有效的ISO 639-1代码的完整列表可以在Wikipedia上的http://en.wikipedia.org/wiki/List_of_ISO_639-1_codes上查看。

默认情况下，所有字符和图案都必须支持英语。如果未将特定字符或主题配置为支持mugen.cfg中设置的语言，则该字符或主题将退回到以英语显示。

语言机制

M.U.G.E.N中的语言机制在定义文件中的组级别上运行。组是指以一行开头的文本块，该行的字符串括在方括号（'['amd']'）中。例如：

[Group 1]
Contents of Group 1.
Another line in Group 1.
This is the last line in Group 1.

[Group 2]
This line is in Group 2.

为了将组本地化为目标语言，必须首先创建组的副本，并且目标语言的ISO 639-1代码后跟一个句点('.')，然后加在组名之前。没有语言代码的组称为未修饰的组(undecorated groups)，名称以语言代码开头的组称为本地化组(localized groups)。例如：

[Group A]
This is the original Group A.
This is called an undecorated group.
The target language for undecorated groups must be English.

[ja.Group A]
This is Group A localized for Japanese.
This is called a localized group that will be loaded when the
preferred language is "ja".

每当M.U.G.E.N尝试加载支持此语言机制的组时，加载程序都会首先在文件中扫描本地化的组名。如果找到，将加载该组。如果未找到本地化的组，则加载程序将在文件中扫描未修饰的组名。按照上面的示例，如果首选语言配置为"ja"，则将在尝试加载[Group A]之前尝试加载[ja.Group A]。

通过这种机制，仅需要对需要基于首选语言具有不同内容的组进行本地化。不需要跨语言的不同内容的组可以保持未修饰状态。为了正确显示未修饰组中的字符串，而与首选语言无关，要求用于未修饰组的语言为英语。

某些组无法使用此机制进行本地化。有关详细信息，请参见"不受本地化限制的组"部分。

本地化说明

  • 本地化主题时，可能需要本地化 system.def 中的[Files]组，以便用适当的胜利引号替换目标语言。包含非ASCII字符的文本需要True Type字体。
  • 本地化情节提要板场景时，本地化组名的未修饰部分必须与其英语等效的组名，例如， [Scene 3 hello]将在西班牙语中定位为[es.Scene 3 hello]，而不是[es.Scene 3 hola]。
  • 建议（但不是必需）用英文（更准确地说，使用ASCII字符集）写出角色和舞台的显示名称，以便在不同的图案上正确显示，并在游戏中保持一致的外观。

免于本地化的组

以下组不能通过上述语言机制进行本地化。

  • mugen.cfg中的所有组。
  • 字符命令和状态组，即[Command]，[State ...]和[Statedef ...]
  • 个别背景元素。
